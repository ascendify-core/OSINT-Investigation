## **3.2 GitHub footprint - small company, outsized open-source reach**

Resend's core open-source project, react-email, is disproportionately large relative to the company's size: 19,300+ GitHub stars, over 1,000 forks, 196 outside contributors, and roughly 2 million weekly downloads on npm (up 108% since its previous major release).

- react-email - a component library for building emails with React instead of legacy HTML table layouts
- 9 official SDKs - official libraries in multiple programming languages so developers in any stack can integrate Resend's email API
- MCP support - Resend can be used directly from Cursor, Claude Desktop, or other MCP-compatible AI tools, a very recent kind of integration

_In plain terms: A GitHub footprint with more stars (19,300+) than the entire company has employees (74) is a strong, measurable signal that this company's reputation and adoption is being driven primarily by developers finding and recommending its open-source tools - not by traditional marketing._

_Source: github.com/resend/react-email; resend.com/blog/react-email-6_

# **4\. Metadata - What Files Can Quietly Reveal**

Metadata is hidden information inside a file that isn't part of its visible content. We didn't extract metadata from a specific Resend file for this exercise, but the concept applies directly to what Resend's own product does.

- Resend's core business is sending transactional emails - every email sent through it carries metadata like sender domain, delivery timestamps, open/click tracking, and DNS authentication records (SPF/DKIM)
- The company's own product includes a feature to detect if a sending domain gets flagged on a spam blocklist (DNSBL) - essentially metadata monitoring as a built-in service
- As a general technique, checking a company's public blog post PDFs or downloadable assets for file properties (author, creation software) works the same way regardless of company size

_In plain terms: Unlike Spotify, where metadata mattered because of royalty disputes, here metadata is quite literally the product: Resend's entire value proposition is managing and interpreting email metadata reliably on a developer's behalf._
